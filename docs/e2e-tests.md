# E2E Tests (Playwright)

The repository includes browser-based E2E tests for the Blazor Server app using Playwright.

Scenarios covered

- CSS isolation applies (grid layout on `.admin-page`).
- Scenario selection interactivity updates the active banner.
- Namespace switch, scope selection, search filtering, and tag toggle.

Running locally

1) Build the solution
- `dotnet build`

2) Install Playwright browsers (once)
- Windows PowerShell script generated by build:
  - `pwsh ./StrataConfig.Tests/bin/Debug/net10.0/playwright.ps1 install`

3) App hosting
- Auto-host: tests boot the Aspire AppHost automatically (no `E2E_BASE_URL` needed).
- Optional manual: you can still run `dotnet run --project StrataConfig.AppHost` and set `E2E_BASE_URL` to override.

4) Run tests
- `dotnet test StrataConfig.Tests -l "console;verbosity=normal"`
- To target an already-running instance, set `E2E_BASE_URL`.

Notes

- If browsers aren’t installed, tests no-op (install via the Playwright script).
- For deterministic test startup, we can add an Aspire.Testing fixture to boot the distributed app and discover the web URL. Ask if you want this wired in.
